    The scripting engine has various predefined object classes - Layers or Maps for example. The pre-defined functions and events for each predefined class are listed below under functions. Objects in all classes must be declared as null in the header:

    

    Global Layer My_Layer;


    Global Map My_Map;
    Moste script objects are non-Instantiable, and must pull an object from the .xml file to be assigned to them using the System function getObject() (without the line break):

    

    My_Layer = System.getContainer("main").


       getLayout("normal").getObject("mylayer");
    where the object "mylayer" is defined in the .xml file like so:

    

    [layer 


      id="mylayer" 


      x="5" y="6" 


      image="player.mylayer" 


    /]





    (replace the ['s with <'s)
    Some objects, however, are Instantiable and can be constructed like this:

    

    My_Map = new Map;
    they can then load the appropriate object using built-in functions, as in Map's loadMap(). The entire list of object classes that can be constructed is:

    Actually, Wac is a bit of an exception. It doesn't need to be constructed, but has to use the System function getWac() - like this (without the line break): 
    

    My_Main_Wac = System.getWac("


      {3CBD4483-DC44-11d3-B608-000086340885}");
    So are Container and Layout, which must be loaded using the system function getContainer or getLayout() (without the line break in My_Layout = ): 
    

    My_Container = System.getContainer("eq");


    My_Layout = System.getContainer("main").


       getLayout("normal");
    The one object which never has to be declared is System, and you can call any of System's pre-defined functions from within the function of any object at any time.

    Garbage Collecting
    Alpha 666 scripting adds support for garbage collecting; this means you can destroy objects discreetly. Just use the delete command: 
    

    delete my_layer;

    Your Own Classes
    In addition, you can create your own classes which can inherit functions from pre-defined classes. Declare them in the header like this: 
    

    Class Button MyOwnButton;
    and so on...

    Your Own Functions
    You can also define your own functions, declaring them in header: 
    

    Function MyCoolFunction(int x, int y);
    and then define it in the body of the script; 
    

    MyCoolFunction(int x, int y) {


      int n = x * 10;


      if (n < 0) n = 0;


      if (n > 255) n = 255;


      System.setVolume(n);


    }
    If you want your function to retun something, declare it in the header: 
    

    Function int MyCoolFunction(int x, int y);
    and then define it in the body of the script; 
    

    int MyCoolFunction(int x, int y) {


      int n = x * 10;


      if (n < 0) n = 0;


      if (n > 255) n = 255;


      System.setVolume(n);


      return n;


    }

    Inheritance top  
    Concept
    So you've created objects from the predefined classes and started using some functions. But exactly what functions can you use with any given object? Enter the concept of Inheritance. Imagine the predefined classes as children with a common parent - these children inherit behaviour in the form of functions from that parent. And these children can have their own children, who inherit their parent's functions AND their grandparent's functions. All of these kids (classes) all part of a family tree, or hierarchy. The family tree analogy is a good one, because the children in a class "family tree" inherit behaviour, or functions, from any of their predecessors. So child x inherits functions from its immediate parent, that parent's parent, and so on...

    Inheritance in WA3
    Class Object is the grand-daddy in WA3: every single other class inherits functionality from Object, such as the ability to have functions at all ;) They also inherit all of Object's functions. In the WA3 Script Engine, Object has eleven children. And GuiObject, one of those children, has many children of its own, who even have their own children. You can see this is all getting rather complex, so I've prepared a diagram.'

    So AnimatedLayer can use all of Layer's functions, all of GuiObject's functions, AND all of Object's functions. Cool huh? I've listed ALL of the functions any given class can use in the function listing, but some of those functions are inherited and refer you back to the definition of that function on the parent class's .html.
